## 1. 这份资料的系统价值

这份资料不是普通岗位说明，而是 ERP 的真实业务输入。

它说明内容创新部当前核心工作不是单一“内容生产”，而是围绕以下对象反复流转：

- 商品卡
- 切片
- 精剪素材
- 网络素材
- 飞流素材
- 创新素材库
- 千川账户
- 达人账户
- 台账
- 开票
- 充值
- 卡审反馈
- 素材审核
- 低质搬运素材处理

因此 ERP 第一版不能只做驾驶舱和数据看板，必须承载员工每天的操作流。

## 3. 剪辑组工作流

### 罗沛 - 剪辑组长

#### 每日 / 每周工作

1. 每周一接收投手的商品卡和切片精剪需求，并进行任务分配。
2. 每周每人分配 70-75 条精剪任务。
3. 每人每天找 2 个品类、约 20 条网络素材，分割后上传到创新素材库。
4. 收到投手反馈的素材问题后，联系对应剪辑进行修改。
5. 整理素材卡审点。
6. 每周一和投手跟进千川账户精剪素材、飞流素材的跑量情况。
7. 每月底对接投手确认飞流素材数量，月底周五前产出所有飞流素材并上传素材库，上传后告知投手。
8. 每周计算投手上传数量，并跟进剪辑任务。

#### 额外工作

1. 赫本希黑松露牙膏新包装拍摄，6.23 拍摄 90 条牙膏素材，每人 30 条。
2. 6.23 拍摄椰油膏、防晒、小白瓶、面膜，每人 30+ 条视频素材，给到赵总和飞哥。
3. 短视频测试、产品素材拍摄、对接第三方开千川账户。

### 剪辑组员

#### 每日 / 每周工作

1. 每人每天找 2 个品类、约 20 条网络素材，分割后上传创新素材库。
2. 每周每人完成 70-75 条精剪剪辑并上传。

#### 额外工作

1. 新包装拍摄。
2. 产品素材拍摄。
3. 短视频测试。

## 4. 投手组工作流

### 刘秀厅 - 投手组长

#### 每日工作

1. 审核创新盘中的商品卡精剪素材。
2. 有问题时反馈给对应剪辑。
3. 无问题时上传到对应产品计划中，并观察过审情况。
4. 遇到卡审问题时，给智投量提审，并把具体卡审点反馈给剪辑修改。
5. 进入负责的千川账户，处理低质搬运素材的提审或删除。
6. 下午 14:00-18:00 审查对应切片素材。
7. 切片素材有问题时反馈给罗沛，再由罗沛反馈给对应剪辑修改。
8. 素材审核无误后，上传到对应达人账户。
9. 每天筛查低质搬运卡审素材情况，进行提审或删除。
10. 跟进千川账户充值、商品卡对应账户、切片项目开票。

#### 额外工作

1. 每月 1-2 次达人直播投放。
2. 统计新增抖音账号，实名刷粉工作。
3. 短视频测试、产品素材拍摄、对接第三方开千川账户。

### 苑佳乐 / 李树花 - 投手组员

#### 商品卡项目每日工作

1. 早上 9:15 前在巨量引擎工作台导出商品卡千川投放数据。
2. 在 WPS 中把数据列转换为数字。
3. 保存为 CSV。
4. 上传到商品卡核算台账系统。
5. 点击 n8n 请求完成核算。
6. 检查抄送至企业微信 / 飞书群的数据是否异常。
7. 点对点进入账户分析调整。
8. 进入负责的千川账户，处理低质搬运素材的提审或删除。
9. 审核创新盘中的商品卡精剪素材。
10. 有问题反馈给对应剪辑，无问题上传到对应产品计划。
11. 花花负责商品卡账户开票：每周一或周三开票，收发票后打印核对金额并交财务。
12. 投手负责对应账户充值，并记录充值金额。

#### 切片项目每日工作

1. 早上 9:15 前在巨量引擎工作台导出切片千川投放数据。
2. 在 WPS 中把数据列转换为数字。
3. 保存为 CSV。
4. 上传到商品卡核算台账系统。
5. 点击 n8n 请求完成核算。
6. 检查抄送至企业微信 / 飞书群的数据是否异常。
7. 点对点进入账户分析调整。
8. 下午 14:00-18:00 审查对应切片素材。
9. 有问题反馈给罗沛，再由罗沛反馈给对应剪辑修改。
10. 审核无误后上传到对应达人账户。
11. 每天筛查低质搬运卡审素材情况，进行提审或删除。
12. 佳乐负责千川切片账户充值、充值记录更新、挂账跟进、开票跟进。

#### 额外工作

1. 每周三固定进行账户素材标题、商品计划维护清理。
2. 每周五固定更新台账。
3. 每月 15 号左右，投手和赵总一起做上个月业绩核算并交叉验证。
4. 不定时导出店铺后台订单，查看退货率。
5. 每周一投手提出商品卡和切片精剪需求，花花和佳乐汇总需求给罗沛，由罗沛分配剪辑任务。
6. 产品改价格、换新包装、加急制作新图时，投手对应账户产品，花花汇总发到美工群给产品和法务审核；审核通过后设计主管下发任务，花花对接设计，收图后给游伟鹏终审；审核通过后花花上传创新盘，投手上传到各自账户。
7. 短视频测试，每天和队员进行 1 小时素材拍摄，整理文本爆款视频。
8. 偶尔使用飞瓜 / 有米云进行数据分析。

## 6. ERP 页面建议

### 经营驾驶舱

用于查看今日经营和执行状态。

必须展示：

- 今日商品卡销售
- 今日切片销售
- 今日利润 / ROI
- 今日待处理素材问题
- 今日卡审问题
- 今日低质搬运处理数
- 今日剪辑任务完成数
- 今日素材上传数
- 今日台账核算状态
- 今日开票 / 充值待办

### 任务中心

用于员工每天处理自己的任务。

页面：

- 我的今日任务
- 精剪任务
- 素材审核任务
- 素材修改任务
- 账户维护任务
- 台账核算任务
- 开票充值任务

### 剪辑工作台

用于剪辑每天处理生产任务，不承载台账、日报、充值、开票。

核心页面：

- 卡审修改队列：投手卡审失败后，自动同步给投手和对应剪辑；剪辑负责修改素材，投手负责复审和提审。
- 每日剪辑数量：罗沛或组长下发每个人当天应完成的素材数量。
- 绿联云盘上传进度：接入当前使用的绿联云盘，按项目路径、剪辑姓名、文件数量、上传时间确认进度。
- 个人销售归因：通过内置浏览器打开后台，读取素材列表；素材按剪辑姓名命名，系统提取姓名后归因每个剪辑产生的销售额。
- 剪映入口：优先把剪映内置到系统；如果技术上暂时不行，先提供打开剪映 / 跳转剪映项目的按钮。

剪辑不需要做：

- 台账核算
- 商品卡日报 / 切片日报
- 充值记录
- 发票对账

### 素材中心

用于展示历史素材、绿联云盘文件夹、人员归档和素材分类，方便按人、项目、素材类型追溯和复用。

页面：

- 绿联云盘文件夹
- 历史素材记录
- 按人员分类
- 按素材类型分类
- 精剪素材
- 卡审问题素材
- 产品拍摄素材
- 封面图 / 场景图 / 混剪素材

商品卡和切片不再作为左侧独立模块：

- 商品卡 / 切片账户动作归入投手工作台。
- 商品卡 / 切片素材归入素材中心。
- 商品卡 / 切片经营数据归入管理驾驶舱和数据报表。
- 商品卡 / 切片相关费用归入财务模块。

### 账户中心

用于管理千川账户、达人账户、抖音账号等。

页面：

- 全部账户列表
- 账户分类统计
- 千川账户
- 商品卡账户
- 切片账户
- 达人账户 / 抖音账号
- 账户状态、余额、负责人、备注
- 后续通过内置浏览器自动采集登录态、余额和异常状态

### 财务模块

用于看本月总体收入和支出。收入只有卖货收入，拆整体支付金额和有效收入金额；支出按投流、物料采买、账户购买、刷粉费用、其他费用分类，并保留历史记录便于追溯。

页面：

- 本月整体支付金额
- 本月有效收入金额
- 本月总支出
- 支出种类
- 充值提醒
- 发票对账
- 物料购买
- 账户购买
- 刷粉数量
- 财务历史记录

核心动作：

- 查看本月收入：整体支付金额、有效收入金额。
- 查看本月支出：投流、物料采买、账户购买、刷粉费用、其他费用。
- 设置账户充值提醒
- 记录充值金额和到账状态
- 记录物料购买
- 记录账户购买
- 记录刷粉数量
- 核对发票金额
- 保留费用历史记录和负责人

### 数据报表

用于管理核算、日报、周报、月度业绩。

页面：

- 商品卡日报
- 切片日报
- 素材任务日报
- 账户维护日报
- 月度业绩核算
- 老板汇报

### 自动化中心

用于承载已经出现的 n8n 和未来自动化。

页面：

- 台账核算自动化
- 数据导入任务
- 自动日报
- 异常提醒
- 自动化执行日志
- 失败任务人工接管

### 数据源接入层

用于承载真实业务数据来源，不直接做给员工看的操作台，而是给投手工作台、剪辑工作台、财务模块、商品卡中心、切片中心和数据报表提供自动采集、识别、映射和任务推送能力。

关键判断：

- 系统不应该依赖员工手工填表来维持工作台。
- 第一优先级不是“生成表格”，而是通过内置浏览器自动采集千川后台、账户后台、素材审核页面里的真实状态。
- 飞书文档和 WPS 在线表格仍然有价值，但主要作为历史底表、账户归属底表、台账兜底数据源，不应成为员工每天工作的主入口。
- 员工短期继续在原后台里操作，系统通过内置浏览器跟随打开页面、截图、识别、抽取字段、记录动作，再生成投手 / 剪辑 / 财务待办。

当前可接入来源：

- 内置浏览器：千川账户后台、账户状态页、计划 / 素材 / 卡审相关页面。
- 截图识别 / 页面文本抽取：余额、账户状态、ROI、卡审提示、低质搬运、素材审核状态。
- 飞书：璟美空间检查分析各类表格
- 飞书：商品卡核算
- 飞书：切片核算
- WPS / 金山文档：抖音千川账户登记表3.13，作为账户归属和负责人兜底底表。

已读取真实数据：

- `登记表0313`：108 条账户记录，其中刘秀厅 / 苑佳乐 / 李树花相关账户 99 条，已生成投手账户底表。
- `排查明细`：97 条排查记录，后续用于补充账户异常、注销、退款、赠款和排查状态。

已访问飞书真实数据源：

- `商品卡核算` 文件夹：商品卡核算台账、店铺退款率、千川商品卡输入表。
- `切片核算` 文件夹：切片核算台账、达人昵称库、千川计划、日系订单、泰系订单。
- `璟美空间 / 2026发财小组` 文件夹：TK内容分发项目、达人切片分发项目、商品卡项目、创新总控看板、九店商品卡周报监控表、项目总控看板。

已读到的飞书表结构：

- 商品卡核算台账：源文件上传记录、核算结果留存、商品卡核算触发 n8n；可见字段包括核算批次、店铺明细 JSON、核算时间、GSV ROI。
- 商品卡店铺退款率：月份、各店铺退款率。
- 切片核算台账：源文件上传记录、核算结果留存、运营人业绩明细、达人业绩明细、切片核算触发 n8n。
- 切片达人昵称库：运营人、归属主达人、达人昵称、达人佣金、产品备注。

数据源映射判断：

- 内置浏览器千川后台：进入投手工作台、账户中心、素材中心、自动化中心。
- 截图 / OCR / 页面文本抽取：生成账户异常、余额提醒、ROI 异常、卡审转交、低质搬运处理任务。
- 卡审失败素材：同步推送给投手和对应剪辑；投手负责判断和提审，剪辑负责修改素材。
- 员工在内置浏览器里的动作：记录打开账户、上传素材、提审、删除、充值确认、备注等操作，沉淀为日报和操作日志。
- 璟美空间检查分析表：主要进入管理驾驶舱、投手工作台、素材中心，用于发现账号、素材、检查异常。
- 商品卡核算：进入商品卡中心、财务模块、数据报表。
- 切片核算：进入切片中心、财务模块、数据报表。
- 抖音千川账户登记表3.13：进入账户归属底表，只作为投手负责人、商品卡 / 切片分类和历史账户信息兜底。

接入优先级：

- P0：内置浏览器自动采集 -> 账户实时状态表、投手今日任务、素材卡审转交任务。
- P0：自动推送规则 -> 投手上班后收到自己的账户异常、低余额、ROI 异常、卡审失败；剪辑同步收到对应素材修改任务。
- P0：WPS 千川账户登记表 -> 账户归属底表，当前已读取 99 条投手相关账户，仅用于初始化和兜底。
- P1：飞书商品卡核算 -> 商品卡日报 / 财务台账，当前已访问台账，待接全量行数据。
- P1：飞书切片核算 -> 切片日报 / 财务台账，当前已访问台账，待接全量行数据。
- P2：飞书璟美空间检查分析 -> 异常中心 / 管理驾驶舱，当前已访问文件列表，待深入读取表格。

自动采集实现判断：

1. 桌面端内置浏览器保存员工登录态。
2. 系统按账户清单逐个打开千川后台页面。
3. 每个页面自动截图，并在固定区域做 OCR / 视觉识别 / 页面文本抽取。
4. 根据规则抽取余额、账户状态、ROI、卡审提示、低质搬运、素材状态等字段。
5. 规则命中后自动生成任务：
   - 低余额 -> 推送投手确认充值。
   - 账户异常 -> 推送投手核对状态。
   - ROI 异常 -> 推送投手进入账户调控。
   - 卡审失败 -> 同步推送投手和对应剪辑。
   - 低质搬运 -> 推送投手提审或删除。
6. 员工在内置浏览器操作时，系统记录动作、截图、时间、账户、结果，沉淀到日报和操作日志。

API 判断：

- 平台 API 如果难接，不作为第一版主路径。
- 内置浏览器采集虽然不如 API 标准，但更贴近员工真实操作，也更容易先跑通。
- 后续如果某个平台 API 可接，再把同一套字段映射切换到 API，不改变工作台任务逻辑。

第一张真正要整理的底表是投手账户表：

- 账号名
- 登记人
- 项目
- 商品卡 / 切片
- 充值状态
- 开票状态
- 当前异常
- 处理状态

## 7. 第一版 P0 功能

P0 不应该追求完整 ERP，而应该先覆盖真实高频动作。

### P0-1 员工任务中心

每个人登录后看到：

- 今日要完成多少精剪
- 今日要审核哪些素材
- 今日要维护哪些账户
- 今日要完成哪些核算
- 今日有哪些卡审 / 异常反馈

### P0-2 素材审核与反馈流

必须打通：

投手审核素材 -> 有问题反馈 -> 罗沛分配修改 -> 剪辑修改 -> 投手复审 -> 上传账户

### P0-3 精剪需求和任务分配

必须打通：

投手提出需求 -> 花花 / 佳乐汇总 -> 罗沛分配 -> 剪辑执行 -> 上传素材库 -> 投手审核

### P0-4 台账核算流程

必须承载：

巨量引擎导出 -> WPS 处理 -> CSV 上传 -> n8n 核算 -> 群消息检查 -> 异常处理

### P0-5 账户维护

必须承载：

- 千川账户维护
- 低质搬运素材提审 / 删除
- 充值记录
- 开票记录

账户维护不能只按“投手组”汇总，必须先按投手拆分，再按账户类型拆分：

- 刘秀厅
  - 商品卡账户
  - 切片账户
- 苑佳乐
  - 商品卡账户
  - 切片账户
- 李树花
  - 商品卡账户
  - 切片账户

每个账户至少要能看到：

- 今日异常
- 当前动作
- 后台入口
- 充值状态
- 开票 / 发票状态
- 素材审核或低质搬运处理状态

## 8. 关键判断

这份资料说明，内容创新部 ERP 的第一版核心不应该只是“看板驾驶舱”。

真正的核心应该是：

1. 任务中心
2. 素材流转
3. 审核反馈
4. 台账核算
5. 账户维护
6. 经营驾驶舱

驾驶舱负责看见问题，任务中心和流程中心负责处理问题。
