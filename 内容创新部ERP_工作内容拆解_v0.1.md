# 内容创新部 ERP 工作内容拆解 v0.1

来源：/Users/Admin/Desktop/内容创新部工作描述.pdf

## 1. 这份资料的系统价值

这份资料不是普通岗位说明，而是 ERP 的真实业务输入。

它说明内容创新部当前核心工作不是单一“内容生产”，而是围绕以下对象反复流转：

- 商品卡
- 切片
- 精剪素材
- 网络素材
- 飞流素材
- 创新素材库
- 千川账户
- 达人账户
- 台账
- 开票
- 充值
- 卡审反馈
- 素材审核
- 低质搬运素材处理

因此 ERP 第一版不能只做驾驶舱和数据看板，必须承载员工每天的操作流。

## 3. 剪辑组工作流

### 罗沛 - 剪辑组长

#### 每日 / 每周工作

1. 每周一接收投手的商品卡和切片精剪需求，并进行任务分配。
2. 每周每人分配 70-75 条精剪任务。
3. 每人每天找 2 个品类、约 20 条网络素材，分割后上传到创新素材库。
4. 收到投手反馈的素材问题后，联系对应剪辑进行修改。
5. 整理素材卡审点。
6. 每周一和投手跟进千川账户精剪素材、飞流素材的跑量情况。
7. 每月底对接投手确认飞流素材数量，月底周五前产出所有飞流素材并上传素材库，上传后告知投手。
8. 每周计算投手上传数量，并跟进剪辑任务。

#### 额外工作

1. 赫本希黑松露牙膏新包装拍摄，6.23 拍摄 90 条牙膏素材，每人 30 条。
2. 6.23 拍摄椰油膏、防晒、小白瓶、面膜，每人 30+ 条视频素材，给到赵总和飞哥。
3. 短视频测试、产品素材拍摄、对接第三方开千川账户。

### 剪辑组员

#### 每日 / 每周工作

1. 每人每天找 2 个品类、约 20 条网络素材，分割后上传创新素材库。
2. 每周每人完成 70-75 条精剪剪辑并上传。

#### 额外工作

1. 新包装拍摄。
2. 产品素材拍摄。
3. 短视频测试。

## 4. 投手组工作流

### 刘秀厅 - 投手组长

#### 每日工作

1. 审核创新盘中的商品卡精剪素材。
2. 有问题时反馈给对应剪辑。
3. 无问题时上传到对应产品计划中，并观察过审情况。
4. 遇到卡审问题时，给智投量提审，并把具体卡审点反馈给剪辑修改。
5. 进入负责的千川账户，处理低质搬运素材的提审或删除。
6. 下午 14:00-18:00 审查对应切片素材。
7. 切片素材有问题时反馈给罗沛，再由罗沛反馈给对应剪辑修改。
8. 素材审核无误后，上传到对应达人账户。
9. 每天筛查低质搬运卡审素材情况，进行提审或删除。
10. 跟进千川账户充值、商品卡对应账户、切片项目开票。

#### 额外工作

1. 每月 1-2 次达人直播投放。
2. 统计新增抖音账号，实名刷粉工作。
3. 短视频测试、产品素材拍摄、对接第三方开千川账户。

### 苑佳乐 / 李树花 - 投手组员

#### 商品卡项目每日工作

1. 早上 9:15 前在巨量引擎工作台导出商品卡千川投放数据。
2. 在 WPS 中把数据列转换为数字。
3. 保存为 CSV。
4. 上传到商品卡核算台账系统。
5. 点击 n8n 请求完成核算。
6. 检查抄送至企业微信 / 飞书群的数据是否异常。
7. 点对点进入账户分析调整。
8. 进入负责的千川账户，处理低质搬运素材的提审或删除。
9. 审核创新盘中的商品卡精剪素材。
10. 有问题反馈给对应剪辑，无问题上传到对应产品计划。
11. 花花负责商品卡账户开票：每周一或周三开票，收发票后打印核对金额并交财务。
12. 投手负责对应账户充值，并记录充值金额。

#### 切片项目每日工作

1. 早上 9:15 前在巨量引擎工作台导出切片千川投放数据。
2. 在 WPS 中把数据列转换为数字。
3. 保存为 CSV。
4. 上传到商品卡核算台账系统。
5. 点击 n8n 请求完成核算。
6. 检查抄送至企业微信 / 飞书群的数据是否异常。
7. 点对点进入账户分析调整。
8. 下午 14:00-18:00 审查对应切片素材。
9. 有问题反馈给罗沛，再由罗沛反馈给对应剪辑修改。
10. 审核无误后上传到对应达人账户。
11. 每天筛查低质搬运卡审素材情况，进行提审或删除。
12. 佳乐负责千川切片账户充值、充值记录更新、挂账跟进、开票跟进。

#### 额外工作

1. 每周三固定进行账户素材标题、商品计划维护清理。
2. 每周五固定更新台账。
3. 每月 15 号左右，投手和赵总一起做上个月业绩核算并交叉验证。
4. 不定时导出店铺后台订单，查看退货率。
5. 每周一投手提出商品卡和切片精剪需求，花花和佳乐汇总需求给罗沛，由罗沛分配剪辑任务。
6. 产品改价格、换新包装、加急制作新图时，投手对应账户产品，花花汇总发到美工群给产品和法务审核；审核通过后设计主管下发任务，花花对接设计，收图后给游伟鹏终审；审核通过后花花上传创新盘，投手上传到各自账户。
7. 短视频测试，每天和队员进行 1 小时素材拍摄，整理文本爆款视频。
8. 偶尔使用飞瓜 / 有米云进行数据分析。

## 6. ERP 页面建议

### 经营驾驶舱

用于查看今日经营和执行状态。

必须展示：

- 今日商品卡销售
- 今日切片销售
- 今日利润 / ROI
- 今日待处理素材问题
- 今日卡审问题
- 今日低质搬运处理数
- 今日剪辑任务完成数
- 今日素材上传数
- 今日台账核算状态
- 今日开票 / 充值待办

### 任务中心

用于员工每天处理自己的任务。

页面：

- 我的今日任务
- 精剪任务
- 素材审核任务
- 素材修改任务
- 账户维护任务
- 台账核算任务
- 开票充值任务

### 素材中心

用于管理素材从收集到审核再到上传的全流程。

页面：

- 创新素材库
- 网络素材收集
- 精剪素材管理
- 飞流素材管理
- 产品拍摄素材
- 卡审问题素材
- 低质搬运素材

### 商品卡中心

用于管理商品卡相关投放、素材、数据和台账。

页面：

- 商品卡账户
- 商品卡素材审核
- 商品卡计划上传
- 商品卡台账核算
- 商品卡开票充值
- 商品卡退货率

### 切片中心

用于管理达人切片素材、达人账户和切片数据。

页面：

- 切片账户
- 达人账户
- 切片素材审核
- 切片素材上传
- 切片台账核算
- 切片开票充值

### 账户中心

用于管理千川账户、达人账户、抖音账号等。

页面：

- 千川账户
- 达人账户
- 抖音账号
- 账户充值记录
- 账户开票记录
- 账户素材维护

### 财务模块

用于承载会直接影响投手正常跑账户的财务动作。财务模块不能只藏在账户中心，因为台账、充值提醒、发票对账一旦延迟，会直接造成账户余额不足、挂账不清、开票异常或投放中断。

页面：

- 台账
- 充值提醒
- 发票对账

核心动作：

- 导入台账数据
- 核对收入、成本、退款、佣金和利润
- 设置账户充值提醒
- 记录充值金额和到账状态
- 核对发票金额
- 标记收票、打印、交财务和异常差额

### 数据报表

用于管理核算、日报、周报、月度业绩。

页面：

- 商品卡日报
- 切片日报
- 素材任务日报
- 账户维护日报
- 月度业绩核算
- 老板汇报

### 自动化中心

用于承载已经出现的 n8n 和未来自动化。

页面：

- 台账核算自动化
- 数据导入任务
- 自动日报
- 异常提醒
- 自动化执行日志
- 失败任务人工接管

### 后台数据源

用于承载真实业务表格来源，不直接做给员工看的操作台，而是给投手工作台、财务模块、商品卡中心、切片中心和数据报表提供底表。

当前可接入来源：

- 飞书：璟美空间检查分析各类表格
- 飞书：商品卡核算
- 飞书：切片核算
- WPS / 金山文档：抖音千川账户登记表3.13

数据源映射判断：

- 璟美空间检查分析表：主要进入管理驾驶舱、投手工作台、素材中心，用于发现账号、素材、检查异常。
- 商品卡核算：进入商品卡中心、财务模块、数据报表。
- 切片核算：进入切片中心、财务模块、数据报表。
- 抖音千川账户登记表3.13：进入投手工作台、账户中心、财务模块。

第一张真正要整理的底表是投手账户表：

- 账号名
- 登记人
- 项目
- 商品卡 / 切片
- 充值状态
- 开票状态
- 当前异常
- 处理状态

## 7. 第一版 P0 功能

P0 不应该追求完整 ERP，而应该先覆盖真实高频动作。

### P0-1 员工任务中心

每个人登录后看到：

- 今日要完成多少精剪
- 今日要审核哪些素材
- 今日要维护哪些账户
- 今日要完成哪些核算
- 今日有哪些卡审 / 异常反馈

### P0-2 素材审核与反馈流

必须打通：

投手审核素材 -> 有问题反馈 -> 罗沛分配修改 -> 剪辑修改 -> 投手复审 -> 上传账户

### P0-3 精剪需求和任务分配

必须打通：

投手提出需求 -> 花花 / 佳乐汇总 -> 罗沛分配 -> 剪辑执行 -> 上传素材库 -> 投手审核

### P0-4 台账核算流程

必须承载：

巨量引擎导出 -> WPS 处理 -> CSV 上传 -> n8n 核算 -> 群消息检查 -> 异常处理

### P0-5 账户维护

必须承载：

- 千川账户维护
- 低质搬运素材提审 / 删除
- 充值记录
- 开票记录

账户维护不能只按“投手组”汇总，必须先按投手拆分，再按账户类型拆分：

- 刘秀厅
  - 商品卡账户
  - 切片账户
- 苑佳乐
  - 商品卡账户
  - 切片账户
- 李树花
  - 商品卡账户
  - 切片账户

每个账户至少要能看到：

- 今日异常
- 当前动作
- 后台入口
- 充值状态
- 开票 / 发票状态
- 素材审核或低质搬运处理状态

## 8. 关键判断

这份资料说明，内容创新部 ERP 的第一版核心不应该只是“看板驾驶舱”。

真正的核心应该是：

1. 任务中心
2. 素材流转
3. 审核反馈
4. 台账核算
5. 账户维护
6. 经营驾驶舱

驾驶舱负责看见问题，任务中心和流程中心负责处理问题。
